🧭 Lab Map: Navigating the AI/ML Learning Lab

This page explains the structure, purpose, and flow of your lab — from homepage to diagnostics.

📄 Core Routes

  • / — Homepage with intro, links, and Copilot widget
  • /lab-log — Renders lab_log.md as a styled reflection journal
  • /tools_stories_index — Auto-generated list of tool tutorials
  • /tool-viewer/<name> — Reads and renders a specific tool story
  • /copilot-widget-query — Handles smart replies for the floating assistant
  • /launch-status — Shows route health, tunnel status, and widget config
  • /md-viewer/<name> — Renders any Markdown file from /docs/
  • /backup-status — Confirms presence of key files and backups
  • /featured-tool — Highlights one tool with summary and link

📁 Folder Structure

  • /templates/ — Jinja2 HTML templates for each route
  • /docs/ — Markdown files for logs, stories, and notes
  • /static/ — CSS and widget assets
  • app.py — Main Flask app with all route logic

🧠 How It All Connects

The homepage links to your log, tool index, and diagnostics. Each tool story is browsable via /tool-viewer/, and the widget replies based on keywords. The lab is modular, readable, and scalable — designed for clarity and growth.
